# 第4课 让任务变成图：构建多轮对话的增量 Logic Graph

#### 一、本讲定位

第3讲已经确立：

> Logic Graph 不是还原 AI 思维链，而是记录 Agent 任务过程中的外部可验证依赖。
> 

第4讲要做的事，修正为：

> **一个任务 = 一张持续更新的图；每轮对话 = 一次 Graph Patch。**
> 

本讲的目标不是造完美系统，而是让学生亲手跑通一个最小的多轮依赖图 PoC。

---

#### 二、本讲核心目标

让学生理解并跑通：
用户多轮对话 → Graph Extractor → Graph Patch → TaskGraph → Mermaid 图

学生学完后要能回答：

1. **什么时候开始建图？**
    
    识别到一个持续任务时开始建图。
    
2. **每轮对话产生什么？**
    
    不是新图，而是一个 `Graph Patch`。
    
3. **Patch 从哪里来？**
    
    由专门的 `Graph Extractor Prompt` 从用户原话中抽取。
    
4. **Python 代码做什么？**
    
    合并 Patch，维护 TaskGraph，输出 Mermaid。

#### 五、第0节：开场回顾【5分钟】

老师先问学生：

> 如果每一轮对话都建一张新图，会有什么问题？
> 

引导学生说出：

```
问题1：一轮对话没有任务全貌
问题2：相邻两轮之间大量重复
问题3：用户不知道什么时候该启动
问题4：图越建越多，管理混乱
```

然后在黑板上写下本讲结论：

```
一个任务 = 一张图
每一轮对话 = 一次补丁（Patch）
补丁只记录变化，不重建全图
```

#### 六、第1节：任务级图 vs 轮次级图【10分钟】

#### 错误模型

```
Turn1 → Graph1
Turn2 → Graph2   （大量重复 Graph1 的内容）
Turn3 → Graph3   （又重复前两张）
```

#### 正确模型

```
Task
  ├── TaskGraph（持续存在，持续更新）
  │
  ├── Turn1 → Patch1 → 合并进 TaskGraph
  ├── Turn2 → Patch2 → 合并进 TaskGraph
  └── Turn3 → Patch3 → 合并进 TaskGraph
```

每一轮对话产生的补丁只包含三种东西：

```
new_nodes         本轮新出现的依赖
updated_nodes     已有节点的内容更新
superseded_nodes  被推翻、被替换的旧节点
```

老师强调：

> Logic Graph 的价值不在"建图"，而在"演化"。能看到任务一步步长出来，才是 Agent 的真实工作轨迹。
>

#### 八、第3节：Graph Patch 数据结构【10分钟】

#### 节点结构

```json
{
  "node_id": "n_0007",
  "type": "Constraint",
  "content": "孩子没有管理员权限，不能用客户端输入命令",
  "created_at_turn": 2,
  "status": "active"
}
```

`status` 只有三种值：`active` / `superseded` / `done`。

#### Patch 结构

```json
{
  "task_id": "task_001",
  "turn_id": 3,
  "new_nodes": [ ... ],
  "updated_nodes": [ ... ],
  "superseded_nodes": [
    {"node_id": "n_0003", "reason": "原方案被推翻"}
  ],
  "new_edges": [ ... ]
}
```

#### 节点类型只保留七种

```
UserGoal
Constraint
Fact
ToolResult
Decision
FileArtifact
OpenTask
```

↓

#### Graph Extractor Prompt【新增】

课堂上先给学生这个模板：

```python
你是 Logic Graph Extractor。

你的任务不是回答用户，也不是解决用户问题。
你的任务是从本轮用户输入中抽取 TaskGraph 的增量补丁 Graph Patch。

你必须只输出合法 JSON，不要输出解释。

节点类型只能是：
UserGoal, Constraint, Fact, ToolResult, Decision, FileArtifact, OpenTask

Patch 格式如下：

{
  "task_id": "...",
  "turn_id": 1,
  "new_nodes": [],
  "updated_nodes": [],
  "superseded_nodes": [],
  "new_edges": []
}

规则：
1. 只抽取本轮新增、变化、被推翻的信息。
2. 不要重复创建已有节点。
3. 如果本轮推翻旧方案，把旧节点写入 superseded_nodes。
4. 如果没有某类变化，对应数组留空。
5. 不要执行用户请求。
6. 不要给建议。
7. 只输出 JSON。
```

#### 十、第5节：最小可运行代码【25分钟】

文件名：`logic_graph_task.py`

```python
"""
注意：
本文件不负责把自然语言转换成 Graph Patch。
自然语言到 Patch 的转换由 Graph Extractor Prompt + LLM 完成。

本文件只做三件事：
1. 接收 patch1 / patch2 / patch3
2. 合并进 TaskGraph
3. 输出 Mermaid 图
"""

import json
from dataclasses import dataclass, field, asdict
from typing import List, Dict, Optional

@dataclass
class Node:
    node_id: str
    type: str
    content: str
    created_at_turn: int
    status: str = "active"   # active / superseded / done

@dataclass
class Edge:
    source: str
    target: str
    relation: str

@dataclass
class TaskGraph:
    task_id: str
    nodes: Dict[str, Node] = field(default_factory=dict)
    edges: List[Edge] = field(default_factory=list)
    turn_counter: int = 0

    def next_node_id(self) -> str:
        return f"n_{len(self.nodes) + 1:04d}"

class GraphStore:
    """内存版任务图存储"""
    def __init__(self):
        self.graphs: Dict[str, TaskGraph] = {}

    def create(self, task_id: str) -> TaskGraph:
        g = TaskGraph(task_id=task_id)
        self.graphs[task_id] = g
        return g

    def get(self, task_id: str) -> Optional[TaskGraph]:
        return self.graphs.get(task_id)

class Deduplicator:
    """简单基于 type + content 归一化的去重"""
    @staticmethod
    def find_existing(graph: TaskGraph, node_type: str, content: str) -> Optional[str]:
        key = (node_type, content.strip().lower())
        for nid, node in graph.nodes.items():
            if (node.type, node.content.strip().lower()) == key \
               and node.status == "active":
                return nid
        return None

class GraphUpdater:
    """把 Patch 合并进 TaskGraph"""
    @staticmethod
    def apply_patch(graph: TaskGraph, patch: Dict):
        turn = patch["turn_id"]
        graph.turn_counter = max(graph.turn_counter, turn)

        # 1. 处理 superseded
        for item in patch.get("superseded_nodes", []):
            nid = item["node_id"]
            if nid in graph.nodes:
                graph.nodes[nid].status = "superseded"

        # 2. 处理 new_nodes（带去重）
        id_remap = {}
        for n in patch.get("new_nodes", []):
            existing = Deduplicator.find_existing(graph, n["type"], n["content"])
            if existing:
                id_remap[n["node_id"]] = existing
                continue
            new_id = n["node_id"] if n["node_id"] not in graph.nodes \
                     else graph.next_node_id()
            id_remap[n["node_id"]] = new_id
            graph.nodes[new_id] = Node(
                node_id=new_id,
                type=n["type"],
                content=n["content"],
                created_at_turn=turn,
            )

        # 3. 处理 updated_nodes
        for u in patch.get("updated_nodes", []):
            if u["node_id"] in graph.nodes:
                graph.nodes[u["node_id"]].content = u["content"]

        # 4. 处理 new_edges（使用 id 重映射）
        for e in patch.get("new_edges", []):
            src = id_remap.get(e["source"], e["source"])
            tgt = id_remap.get(e["target"], e["target"])
            graph.edges.append(Edge(src, tgt, e["relation"]))

def to_mermaid(graph: TaskGraph) -> str:
    lines = ["graph TD"]
    for nid, node in graph.nodes.items():
        label = f"{node.type}\\n{node.content[:20]}"
        if node.status == "superseded":
            lines.append(f'    {nid}["❌ {label}"]:::dead')
        else:
            lines.append(f'    {nid}["{label}"]')
    for e in graph.edges:
        lines.append(f'    {e.source} -->|{e.relation}| {e.target}')
    lines.append("    classDef dead fill:#fdd,stroke:#c33;")
    return "\n".join(lines)

# ---------------- 课堂演示 ----------------
if __name__ == "__main__":
    store = GraphStore()
    graph = store.create("task_mc_001")

    patch1 = {
        "task_id": "task_mc_001", "turn_id": 1,
        "new_nodes": [
            {"node_id": "n_0001", "type": "UserGoal",
             "content": "构建 Minecraft AI Agent"},
            {"node_id": "n_0002", "type": "UserGoal",
             "content": "支持中文自然语言控制游戏"},
            {"node_id": "n_0003", "type": "Fact",
             "content": "示例指令：给我一把钻石剑、传送到村庄"},
        ],
        "new_edges": [
            {"source": "n_0002", "target": "n_0001", "relation": "refines"},
        ],
    }

    patch2 = {
        "task_id": "task_mc_001", "turn_id": 2,
        "new_nodes": [
            {"node_id": "n_0004", "type": "Decision",
             "content": "采用命令映射器方案"},
            {"node_id": "n_0005", "type": "Decision",
             "content": "通过客户端斜杠命令执行"},
        ],
        "new_edges": [
            {"source": "n_0004", "target": "n_0002", "relation": "serves"},
            {"source": "n_0005", "target": "n_0004", "relation": "implements"},
        ],
    }

    patch3 = {
        "task_id": "task_mc_001", "turn_id": 3,
        "new_nodes": [
            {"node_id": "n_0006", "type": "Constraint",
             "content": "孩子账号无 OP 权限"},
            {"node_id": "n_0007", "type": "Decision",
             "content": "改为通过 RCON 从服务端执行命令"},
            {"node_id": "n_0008", "type": "OpenTask",
             "content": "接入 RCON 协议并测试 /give 执行"},
        ],
        "superseded_nodes": [
            {"node_id": "n_0005",
             "reason": "客户端方案因权限问题不可行"},
        ],
        "new_edges": [
            {"source": "n_0006", "target": "n_0005", "relation": "invalidates"},
            {"source": "n_0007", "target": "n_0004", "relation": "implements"},
            {"source": "n_0008", "target": "n_0007", "relation": "depends_on"},
        ],
    }

    for p in [patch1, patch2, patch3]:
        GraphUpdater.apply_patch(graph, p)
        print(f"\n====== After Turn {p['turn_id']} ======")
        print(f"Nodes: {len(graph.nodes)}  Edges: {len(graph.edges)}")

    print("\n====== Final Mermaid ======")
    print(to_mermaid(graph))
```

```
✅ 第 1 轮：提出目标
✅ 第 2 轮：做出决策
✅ 第 3 轮：推翻前一个决策，换新方案
```

然后完成：

1. 写出 `patch1` / `patch2` / `patch3` 三个 JSON。
2. 放进上面代码运行。
3. 导出 Mermaid，截图给老师。

评分标准：

```
[1 分] 三个 Patch 结构正确
[1 分] 第 3 轮正确使用 superseded_nodes
[1 分] 节点之间有合理的边
[1 分] 图中能直观看到"被推翻"的节点
```

---

#### 十二、本讲老师必须反复强调的三句话

1. **一个任务一张图**，不是一轮对话一张图。
2. **每轮只做 Patch**，不做全量重建。
3. **被推翻的节点不删**，用 `superseded` 留下任务演化轨迹。
