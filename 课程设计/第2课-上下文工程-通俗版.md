
### 第2课教案：AI的“脑容量”危机——Claude Code如何记住整本书

| 项目 | 内容 |
| :--- | :--- |
| **课题** | AI的“脑容量”危机——Claude Code如何记住整本书 |
| **授课对象** | 初中生（对AI感兴趣，有过与ChatGPT/Claude对话体验） |
| **课时** | 45分钟 |
| **对应课程总表** | 第2讲：顶级Agent的战场：上下文工程 |
| **Claude Code 线索** | `REACTIVE_COMPACT`, `CONTEXT_COLLAPSE`, `TOKEN_BUDGET` |
| **Logic Graph 回应** | 引入 `ReasoningNode`，理解为什么“按时间删”不如“按逻辑留” |
| **核心输出物** | 一张手绘对比图：“垃圾桶记忆法” vs “侦探线索墙记忆法” |

---

#### 一、教学目标

1. **知道**：能说出AI的“工作记忆”（上下文窗口）是有限的，就像一张小书桌。
2. **理解**：能解释Claude Code用了哪些“断舍离”招数来防止脑容量爆炸（压缩、截断、预算提醒）。
3. **迁移**：能意识到“保留逻辑关系”比“记住所有流水账”更聪明，并尝试用箭头画出一个简单的“依赖关系图”。

---

#### 二、教学重点与难点

- **重点**：用“书桌/行李箱”的比喻讲清楚Token预算的概念，以及“装不下时怎么办”的几种策略。
- **难点**：理解“依赖图”的抽象概念——为什么删掉最早的话，会让AI的逻辑断掉。

---

#### 三、教学准备

- **教具**：
  - 一张A3纸，画一个**行李箱**（代表上下文窗口，容量200个格子）。
  - 十几张便利贴，写上不同内容（如“任务：写一篇作文”、“开头段落”、“查资料结果1”、“查资料结果2”……）。
  - 一根长绳子和几根短绳子。
- **板书/PPT**：准备一张五层防御塔的图（像游戏里的防御塔一样），标注“第一层：少说废话”、“第二层：揉成纸团”等趣味名称。

---

#### 四、教学过程

| 环节 | 时间 | 教师活动 | 学生活动 | 设计意图 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 导入：AI的“脑瓜崩”时刻** | 5 min | **情景剧**：“假设你让AI帮你写一本小说。你让它先读前10章找伏笔，它读了。你又让它查20个成语，它查了。最后你说‘好了，根据前面的伏笔写第11章吧！’ 结果AI说：‘对不起，前面的伏笔是什么来着？我忘了……’”<br><br>**提问**：AI是真傻吗？还是遇到了什么物理限制？引出概念——**AI的工作记忆是有容量的**，就像你的书桌只能摊开3本书，摊第4本就得把一本合上塞进抽屉。 | 笑，共鸣，回答“因为它记性不好/内存不够”。 | 用共情的失败场景激发好奇心。 |
| **2. 新授：破解脑容量的“五层防御塔”** | 15 min | **A. 核心规则（行李箱游戏）**：<br>1. 讲解Token（把字切成小方块，AI一次只能看20万个方块）。<br>2. 请两位同学上台，把代表“系统设定”、“对话历史”、“工具结果”的便利贴往行李箱里塞，直到塞不下。<br><br>**B. 讲解Claude Code的“五大断舍离绝招”**（对应专业教案的五层防御）：<br>- **第一层：少废话（源头截断）**：如果查资料的结果有10页纸，只把第一段贴进行李箱，剩下的塞进“床底储物箱”（硬盘），告诉AI“东西在床底，需要再拿”。<br>- **第二层：揉成团（Context Collapse）**：AI连续搜了10次文件，把这10次操作揉成一个纸团，写上“搜了10次，没找到想要的文件”。<br>- **第三层：清垃圾（MicroCompact）**：每隔1小时自动把桌面上没用的废纸（过时的临时文件）扔掉。<br>- **第四层：写摘要（AutoCompact）**：行李箱快满了，叫另一个AI助手过来：“把这50条对话总结成100字塞进去！”<br>- **第五层：急救针（Reactive Compact）**：行李箱彻底炸了，AI系统报错。紧急启动，强行把最早的东西撕掉，先活下来再说。 | 参与行李箱填充游戏，感受“空间不足”。听比喻，对应到游戏里的防御塔技能。 | 将五层抽象架构转化为具象的、有画面感的动作（塞、揉、扔、写、撕）。 |
| **3. 探究：为什么“撕掉最早的东西”是大昏招？** | 10 min | **活动：侦探断案——消失的线索**。<br>1. 在黑板上按时间顺序贴便利贴：<br>   [案发地点：图书馆] → [证人A：看见黑影] → [搜查令：查监控] → [发现：黑影是管理员] → [结论：管理员是误会]。<br>2. **模拟最蠢的办法**：老师说“脑容量满了！把最早的两张撕掉！”（撕掉“图书馆”和“证人A”）。<br>3. **提问**：“现在的AI只看到‘搜查令’和‘结论’，它知道为什么要去查监控吗？它知道黑影是谁看见的吗？”<br>4. **揭示真理**：**信息是有骨头的（依赖关系）！** 不能按时间删，要按骨头连着肉的关系删。 | 观察便利贴被撕掉后逻辑断裂。回答：“不知道了，没头没尾的。” | 从体验中得出“依赖关系”的结论，为Logic Graph铺路。 |
| **4. 破局：Logic Graph——给AI画一张“思维导图”** | 10 min | **画图演示**：<br>1. 把刚才撕碎的便利贴重新贴好，然后用粉笔**画箭头**：<br>   [图书馆] ——> [证人A] ——> [搜查令] ——> [结论]<br>2. **讲解**：Claude Code现在的办法是“把前面的写成摘要”（把图书馆和证人揉成一句话），但我们将来想发明更聪明的办法——**Logic Graph（逻辑地图）**。<br>3. **对比**：<br>   - 流水账记忆法：撕掉开头，后面全乱。<br>   - 逻辑地图记忆法：就算我把[图书馆]这张纸条扔了，只要箭头还指着[证人A]，AI就还能推理：“哦，证人A是在某个地方看见的，这个地方不重要，重要的是他看见了。”<br>4. **金句**：**“流水账会忘，但思维导图永远知道路该怎么走。”** | 观察箭头，理解“保留关系网”。尝试自己在本子上画一个类似的“做数学题步骤”的依赖箭头。 | 完成从“源码功能”到“原创理论”的升华，种下图计算的种子。 |
| **5. 小结与作业** | 5 min | **总结**：今天我们知道了AI不是记性差，是书桌太小。Claude Code用了“揉纸团、写摘要”的聪明办法来整理书桌。而我们发现，最厉害的办法是**画地图**。<br><br>**课后小挑战（选做）**：<br>回忆你今天放学回家要做的一件复杂的事（比如：做西红柿炒鸡蛋）。画一张“依赖关系图”：<br>- 圆圈里写步骤（切西红柿、打鸡蛋、开火……）<br>- 用箭头标出“必须先做哪步，才能做哪步”。<br>下节课带来，看看谁画的地图最清晰！ | 记录作业，收拾学具。 | 将抽象的图概念带回生活场景，降低理解门槛。 |

#### 五、板书设计（黑板草图）

```
第2课：AI的“脑容量”危机

[行李箱] = 上下文窗口（200,000个Token格子）
塞入：系统设定(10格) + 历史对话 + 查资料结果(100格) = 满了！

Claude Code的五层防御塔（断舍离大法）：
L1 少废话： 资料太长？ 只贴第一页，其余塞床底。
L2 揉成团： 搜了10次文件？ 揉成一句话“搜过了，没有”。
L3 清垃圾： 1小时前的草稿纸？ 自动碎纸机粉碎。
L4 写摘要： 快满了？ 召唤小弟把50句话总结成3句。
L5 急救针： 炸了？ 强行撕掉最早的话（但线索会断！）。

❌ 笨办法：撕最早的话（线索断了）
✅ 聪明法：画线索地图（Logic Graph）
   [A] → [B] → [C]  （箭头=因果关系）
   就算A丢了，B也知道自己是从哪来的！
```

#### 六、教学后记（教师备忘）

本节课成功的关键在于**行李箱游戏**和**撕便利贴**的视觉冲击。对于初中生，源码里的 `REACTIVE_COMPACT` 是毫无意义的乱码，但“行李箱炸了打急救针”他们能记住一辈子。

**需要警惕的点**：
- 在讲“五层防御”时，语速要慢，每讲一层要停下来让学生复述那个比喻（比如问“第一层叫什么来着？少废话！”）。
- 在讲Logic Graph时，不需要讲节点和边的数据结构，**只要学生能画出带箭头的圆圈**，本节课的目标就完美达成了。那根箭头，就是未来所有复杂图计算的种子。
